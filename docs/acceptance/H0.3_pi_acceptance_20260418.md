## 动机回顾

上一轮 H0 验收（`2f51247`）里 `bind_retry` 失败：两次 `systemctl restart lelamp` 之后 journal 里 `"motor bus port 127.0.0.1:8770 busy; retrying bind for up to 12.0s"` → `"server not started (waited 12.0s)"`，arbiter 整段生命周期失效，dashboard / CLI 被迫回到直连路径。

根因：`_port_is_free()` 探测 socket 没设 `SO_REUSEADDR`；而实际 uvicorn listener 会设。TIME_WAIT 残留在 kernel 里最长存活 `tcp_fin_timeout`（Linux ~60s），导致 probe 在整个 12s 窗口里永远返回 False，尽管 uvicorn 本身可以在毫秒级绑上。

H0.3 做两件事：
1. `_port_is_free` 的 probe socket 现在 `setsockopt(SO_REUSEADDR, 1)`，和 uvicorn 对齐。
2. `bind_retry_total_s` 默认值 `12.0 → 30.0`，覆盖"旧 MainPID 晚几秒才真正释放 listener"的边缘情况。

## 关键路径（Test 4）详细数据

### 施压：seed TIME_WAIT

反复打 `/health` 5 次，让端口 8770 堆积 TIME_WAIT：

```
ss TIME_WAIT count on 8770 before restart: 126
LISTEN    0 2048 127.0.0.1:8770  0.0.0.0:*
TIME-WAIT 0 0    127.0.0.1:8770  127.0.0.1:38238
TIME-WAIT 0 0    127.0.0.1:8770  127.0.0.1:36492
... (共 126 条 TIME_WAIT)
```

126 条 TIME_WAIT，远超旧版 H0 场景的实际情况；旧版只有个位数 TIME_WAIT 就已经跑不起来。

### Back-to-back restart

```
t0 01:04:08.329
t1 01:04:10.354  (first restart returned,  +2.025s)
t2 01:04:10.398  (second restart returned, +0.044s since t1)
```

两次 restart 间隔仅 **44ms**。旧版 H0 在这个场景下必死。

### arbiter 恢复时间线

```
t+02s  HTTP 000  (uvicorn 还没 ready)
t+04s  HTTP 200  {"ok":true,"motor_ok":true,"rgb_ok":true}
t+06s  HTTP 200  {"ok":true,"motor_ok":true,"rgb_ok":true}
t+10s  HTTP 200  {"ok":true,"motor_ok":true,"rgb_ok":true}
t+15s  HTTP 200  {"ok":true,"motor_ok":true,"rgb_ok":true}
```

**第二次 restart 之后 2–4s arbiter 就接管服务**，`motor_ok` / `rgb_ok` 全绿。

### Sentinel 状态

```json
{"pid": 52414, "port": 8770}
MainPID=52399
NRestarts=0
ActiveEnterTimestamp=Sat 2026-04-18 01:04:10 CST
```

sentinel pid 52414 和 systemd MainPID 52399 有小差别，这是 python 父子进程模型（uvicorn 在 worker 线程，sentinel 由上层 python 写）；不影响健康判定。

### Journal: bind 告警清零

```
journalctl -u lelamp.service --since "30 seconds ago" | grep -iE "motor bus|bind|port 877|server not started|retrying"
(empty)
```

**零告警**。说明 SO_REUSEADDR probe 第一次就返回 True，MotorBusServer 根本没进 retry loop。对比旧版的 `"motor bus port 127.0.0.1:8770 busy; retrying bind for up to 12.0s"` → `"server not started"`。

## 对比：旧版 vs 新版（TIME_WAIT 高压下）

| 维度 | 旧版 H0 (`2f51247`) | 新版 H0.3 (`c7d830a`) |
|---|---|---|

| journal 告警 | "busy; retrying..." / "server not started" | 零告警 |
| `/health` motor_ok | 降级 false（无 server） | ✅ true |
