## ⚙️ 硬件配置流程

### 阶段 1：舵机配置（组装前）

**前提：** 需要舵机驱动板和 USB 数据线

```bash
# 1. 进入工作目录
cd /Users/wujiajun/Downloads/Lelamp/lelamp_runtime

# 2. 查找舵机驱动板连接的端口
# 插拔 USB 来识别哪个端口是你的驱动板
uv run lerobot-find-port

# 3. 给每个舵机设置唯一 ID (1-5)
# 按提示依次连接每个舵机并设置 ID
uv run -m lelamp.setup_motors --id my_lamp --port /dev/ttyACM0

# 4. 校准所有舵机
uv run -m lelamp.calibrate --id my_lamp --port /dev/ttyACM0
```

### 阶段 2：硬件组装

参考文档：`docs/3. LeLamp Assembly.md`

**关键步骤：**
1. 安装热熔螺母到 3D 打印件
2. 将已配置 ID 的舵机装入对应位置
3. 连接所有电线
4. 组装完整台灯

### 阶段 3：Raspberry Pi 设置

参考文档：`docs/4. LeLamp Setup.md`

**步骤：**

1. **刷写 Raspberry Pi OS Lite (64-bit)**
   ```bash
   # 使用 Raspberry Pi Imager 工具
   # 选择：Raspberry Pi OS Lite (64-bit)
   # 设置 WiFi、用户名、密码、主机名、启用 SSH
   ```

2. **SSH 连接到 Pi**
   ```bash
   ssh your_username@your_hostname.local
   ```

3. **安装 LeLamp Runtime**
   ```bash
   # 克隆仓库
   git clone https://github.com/humancomputerlab/lelamp_runtime.git
   cd lelamp_runtime

   # 安装 uv
   curl -LsSf https://astral.sh/uv/install.sh | sh

   # 安装音频依赖
   sudo apt-get install portaudio19-dev

   # 安装完整依赖（包含硬件支持）
   uv sync --extra hardware
   ```

4. **配置 ReSpeaker 音频硬件**
   ```bash
   # 下载设备树
   curl https://raw.githubusercontent.com/Seeed-Studio/seeed-linux-dtoverlays/refs/heads/master/overlays/rpi/respeaker-2mic-v2_0-overlay.dts -o respeaker-2mic-v2_0-overlay.dts

   # 编译并安装
   dtc -I dts respeaker-2mic-v2_0-overlay.dts -o respeaker-2mic-v2_0-overlay.dtbo
   sudo dtoverlay respeaker-2mic-v2_0-overlay.dtbo
   sudo cp respeaker-2mic-v2_0-overlay.dtbo /boot/firmware/overlays

   # 编辑 config.txt
   sudo tee -a /boot/firmware/config.txt >/dev/null <<'EOF'
   dtoverlay=respeaker-2mic-v2_0-overlay
   dtoverlay=i2s-mmap
   EOF

   # 配置音频
   sudo tee /etc/asound.conf > /dev/null <<'EOF'
   pcm.!default {
       type plug
       slave {
           pcm "hw:3,0"
       }
   }

   ctl.!default {
       type hw
       card 3
   }
   EOF

   # 重启
   sudo reboot
   ```

5. **测试硬件**
   ```bash
   # 测试 RGB LED
   sudo uv run -m lelamp.test.test_rgb

   # 测试音频
   uv run -m lelamp.test.test_audio

   # 测试舵机
   uv run -m lelamp.test.test_motors --id my_lamp --port /dev/ttyACM0
   ```

## 🎮 控制模式

### Follower 模式
台灯跟随你的手动操作，记录并学习动作。

### Leader 模式
台灯自主执行预设或录制的动作序列。

### 录制和回放

```bash
# 录制一个新动作（命名为 "wave"）
uv run -m lelamp.record --id my_lamp --port /dev/ttyACM0 --name wave

# 回放录制的动作
uv run -m lelamp.replay --id my_lamp --port /dev/ttyACM0 --name wave

# 列出所有录制
uv run -m lelamp.list_recordings --id my_lamp
```

## 🔧 故障排查

### 常见问题

**Q: uv sync 失败，提示 portaudio 错误**
```bash
brew install portaudio  # macOS
sudo apt-get install portaudio19-dev  # Raspberry Pi
```

**Q: 找不到舵机端口**
- 确保舵机驱动板已通过 USB 连接
- 尝试插拔 USB 重新运行 `lerobot-find-port`
- 检查 USB 线是否是数据线而非仅充电线

**Q: 舵机不响应**
- 检查舵机是否正确供电
- 确认舵机 ID 已正确设置（1-5）
- 尝试重新运行校准命令

**Q: 音频设备无法识别**
- 确认 ReSpeaker 驱动已安装
- 运行 `aplay -l` 检查设备列表
- 使用 `alsamixer` 检查音频设置
